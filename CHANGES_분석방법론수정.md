# 분석 방법론 수정 정리 (`fix/analysis-methodology`)

포크(`main` 기준) 이후 대시보드(`app.py`)의 **데이터 사용·분석 방법·시각화·통계적 타당성**을 점검하며
잘못된 분석이나 오해를 부르는 시각화를 바로잡았습니다.
기능 추가가 아니라 **"틀린 것을 고치는"** 작업이 핵심입니다.

아래는 **실제 진행한 순서대로(위 = 먼저, 아래 = 최신)** 커밋 단위로 *문제 → 수정 → 이유*를 정리한 것입니다.

---

## 1. 위험점수 부모 플래그 버그 수정 — `cfd487a`

**문제**
- `위험점수` 계산에서 "부모 비동거"를 `COL_PARENT == 0`으로 판정했는데,
  실제 코드값은 **1(동거) / 2(비동거)**라서 이 조건이 **항상 거짓**이었음.
- 결과적으로 부모와 따로 사는 청년 **291명의 위험요인이 통째로 누락**됨.

**수정**
- 조건을 `COL_PARENT != 1`로 변경해 비동거 청년이 위험요인으로 정상 반영되게 함.

## 2. 페이지2 생활비 충당 Sankey 수정 — `2a19088`

**문제**
- 한 그림 안에서 **월 단위와 연 단위가 섞여** 흐름 크기가 서로 비교 불가능했음.
- 이전소득을 **이중 계산**(총소득에도, 별도 항목에도)해 합이 부풀려짐.
- 들어오는 흐름과 나가는 흐름의 **총량이 맞지 않아(flow imbalance)** Sankey가 왜곡됨.

**수정**
- 단위를 **연 단위로 통일**하고, 청년 연간 총소득을
  **자력소득 / 사적 이전소득 / 공적 이전소득**으로 분해해 이중계산 제거.
- 소득 구성은 Sankey로, "연간 생활비 vs 소득원·부채" 비교는 **별도 막대그래프**로 분리.
- 단위 환산·소득 분해·데이터 한계를 캡션으로 명시.

## 3. 페이지4 히트맵 → 노출 비율 비교로 교체 — `13bcae1`

**문제**
- "공적 이전소득과 부채의 관계"를 **밀도 히트맵**으로 보여줘 마치 관계가 있는 것처럼 읽혔지만,
  실제 Spearman 상관은 **거의 0**이라 근거 없는 시각화였음.

**수정**
- 히트맵을 **"가족지원 없음 집단의 공적지원·부채·고립 노출 비율" 막대그래프**로 교체.
- Spearman 상관계수를 캡션에 동적으로 표시해 **"상관 없음"을 명시**하고 교체 이유를 설명.

## 4. 페이지1 추이 그래프 수정 — `25f138b`

**문제**
- 천단위 콤마가 있는 인구 수치(`1,234`)가 숫자로 파싱되지 않아 **0으로 처리**됨.
- `전체`·`15–64세`·`15–29세`·`15–24세`처럼 **다른 구간과 겹치는 합계 구간**이 함께 그려져
  중복 집계로 추이가 왜곡됨.

**수정**
- 콤마 제거 후 숫자 변환하도록 파싱 로직 수정.
- 서로 겹치지 않는 **단일 연령대만**(15–19 / 20–29 / 30–39 / 40–49 / 50–59 / 60+) 표시.
- 필터 슬라이스에 `.copy()`를 적용해 `SettingWithCopyWarning` 제거.

## 5. 0-팽창 변수: 평균 → 보유율·보유자 중앙값 — `b15ff4c`

**문제**
- 부채·이자·사적/공적 이전소득은 **0인 응답자가 대부분**이라 단순 평균이 심하게 왜곡됨
  (소수의 큰 값이 평균을 끌어올려 "평균 부채 OO만원"이 집단을 대표하지 못함).
- 캡션의 `~`(예: `15~64세`)가 마크다운 취소선으로 해석돼 글자가 깨짐.

**수정**
- 헬퍼 `holding_rate()`(보유율), `holder_median()`(보유자 중앙값) 추가.
- 페이지1 지표: `평균 부채` → **`부채 보유율`** + 보유자 중앙값 캡션.
- 페이지5 표: `평균부채`·`평균이자` → **`부채보유율`·`부채중앙값(보유자)`·`이자보유율`**.
- 페이지6 표: 사적/공적 이전소득·부채·이자 평균 → **각 `보유율`**.
- 캡션의 `~`를 `–`(en dash)로 바꿔 취소선 깨짐 수정(페이지 1·5·6).

## 6. 페이지3 가족 안전망 — 비모수 통계 검정 추가 — `767f2bf`

**문제**
- "부모와 같이 살면 부채/생활비가 다르다", "가족지원 여부로 생활비가 다르다" 같은 주장을
  **그래프(평균 비교)만으로** 제시해 통계적 근거가 없었음.
- 데이터가 0-팽창·비대칭이라 평균/정규분포 가정 검정이 부적절.

**수정** (`scipy.stats` 사용, `requirements.txt`에 `scipy` 추가)
- **부모 동거 ↔ 부채 보유**: 카이제곱 독립성 검정(기대빈도 작으면 Fisher 정확검정).
- **부모 동거 ↔ 부채 총액**: Mann–Whitney U 검정(비모수).
- **가족지원 ↔ 월 생활비**: Mann–Whitney U 검정(비모수).
- 표본이 너무 작으면(n<10) 검정을 건너뛰도록 가드 처리.
- 왜 비모수 검정을 썼는지, 파생변수를 검정에서 제외한 이유를 캡션으로 설명.

> 참고: 카이제곱은 "분포/빈도" 기반이라 정규성 가정이 없는 **비모수 계열**로 분류됩니다.

## 7. 인사이트·정책 시사점 문구 정비 — `b41b7a6`

**문제**

## 8. 페이지5 위험 게이지 교체 + 위험수준 필터 순환참조 해결 — `c6d8c20`

**문제**
- 기존 게이지는 **평균 위험점수(0~5)**를 표시했는데, 저위험(0~1점) 응답자가 워낙 많아
  평균이 0점대로 깔려 "심각성"이 거의 드러나지 않았음.
- 더 큰 버그: 사이드바 **`위험수준` 필터**가 게이지와 **같은 변수(`위험점수`)**를 거르기 때문에
  - `저위험` 선택 → 게이지 **0%** 고정
  - `중위험`/`고위험` 선택 → 게이지 **100%** 고정
  → 표시값이 필터에 의해 결정되는 **순환참조**라 의미가 없었음.

**수정**
- 게이지를 **"위험요인 2개 이상 중첩(중·고위험) 비율"(0~100%)**로 변경.
  - 평균이 가리던 "전체의 약 N%가 위험 중첩"이라는 심각성이 헤드라인으로 드러남.
  - 전체 비율 대비 **±%p** 델타(빨강=더 위험 / 초록=덜 위험)와 기준선 표시.
- 게이지 계산에서만 `위험수준` 필터를 **제외**(`filtered_excl_risk`)해 순환참조 제거.
  - 부모동거·가족지원·생활비 등 **다른 필터에는 정상 반응**.
  - `위험수준` 필터가 켜져 있을 때는 "게이지에는 위험수준 필터 미적용" 안내 캡션 표시.
- 분포 히스토그램·박스플롯·표는 **`위험수준` 필터를 그대로 반영**(역할 분리).
