# 쉬었음 청년은 무엇으로 버티고 있는가?

> 생활비·부채·소득지원으로 본 청년 비경제활동의 **생활안전망 격차** 분석 대시보드

청년 비경제활동(이른바 "쉬었음") 문제를 **규모 통계**에서 멈추지 않고,
"이들이 노동시장 밖에 있는 동안 **무슨 자원으로 생활을 유지하는가**"라는 질문으로 확장해
가족·공공·금융·관계망 등 생활안전망의 내부 격차를 추적하는 Streamlit 대시보드입니다.

---

## 핵심 질문

1. 쉬었음 청년은 얼마나 되고, 어떻게 변해 왔는가? (규모·추이)
2. 이들은 무엇으로 생활비를 버티는가? (소득 구성·부채)
3. 가족 안전망은 실제로 작동하는가? (부모 동거·가족지원)
4. 가족이 없다면 공공·금융·관계망 중 무엇에 기대는가?
5. 누가 가장 위험한가? (위험요인 중첩)
6. 어떤 유형으로 나뉘는가? (생활안전망 유형)
7. 한국 청년의 진짜 안전망은 무엇인가? (종합)

---

## 데이터

SQLite DB(`쉬었음 청년 분석.db`)의 세 테이블을 사용합니다.

| 테이블 | 용도 |
|---|---|
| `EDA용 2024 청년삶실태조사` | 주 분석 데이터(생활비·소득·부채·지원망 등 개인 단위) |
| `연령_활동상태별_쉬었음__비경제활동인구` | 연도·연령대별 쉬었음/비경제활동 추이 |
| `한국노동패널조사` | 보조 패널 자료 |

> 데이터는 KOSSDA 등 공개 조사자료를 가공한 것이며, 본 저장소에는 분석용 가공본만 포함됩니다.

### 단위 주의
- 소득·이전소득은 **연 단위**, 생활비·이자는 **월 단위**로 조사됨 → 비교 시 생활비는 연 환산(월×12).
- 모든 금액은 **1인 평균·만원** 기준.

---

## 파생 변수 정의

| 변수 | 정의 |
|---|---|
| **위험점수** (0–5) | 다음 5개 위험요인의 합: 부모 비동거 · 가족지원 없음 · 도움망 없음 · 부채 보유 · 이자 부담 |
| **위험수준** | 저위험(0–1) / 중위험(2–3) / 고위험(4–5) |
| **생활안전망유형** | 고립위험형 · 금융부담형 · 가족완충형 · 공공지원형 · 대체지원형 · 취약잠재형 (우선순위 규칙으로 분류) |
| **보유율 / 보유자 중앙값** | 0-팽창 변수(부채·이자·이전소득)에서 평균 왜곡을 피하기 위한 지표 |

---

## 분석 방법론 원칙

- **0-팽창(zero-inflated) 변수**: 부채·이자·이전소득은 0인 응답자가 대부분이라
  단순 평균이 소수 고액값에 휘둘림 → **보유율 + 보유자 중앙값**으로 표시.
- **비모수 통계 검정**: 분포가 비대칭이라 정규성 가정 검정 대신
  **Mann–Whitney U**(연속형 비교), **카이제곱 / Fisher 정확검정**(범주형 연관)을 사용.
- **관계 없음(null)도 정직하게**: 상관이 없는 변수쌍은 오해를 부르는 시각화 대신
  2×2 비교·검정으로 "관계 없음"을 근거와 함께 직접 제시.
- **일관성**: 인사이트·정책 문구는 필터를 바꿔도 어긋나지 않도록 정성적으로 서술.

## 페이지 구성

1. **전체 규모와 문제 제기** — 연령대별/청년(15–29세) 쉬었음 추이, 기본 지표
2. **무엇으로 버티는가** — 소득 구성 Sankey(자력/사적이전/공적이전), 생활비 vs 가용자원
3. **가족 안전망** — 부모 동거·가족지원별 비교 + 비모수 통계 검정
4. **가족 밖의 안전망** — 대체 지원망, 공적지원×부채 관계 검정
5. **위험지수 분석** — 위험군 비율 게이지, 위험수준별 분포·프로파일
6. **생활안전망 유형** — 유형별 규모(트리맵)·자원 보유율·평균 위험점수
7. **최종 결론** — 안전망 구성 종합

## 실행 방법

```bash
# 1) 의존성 설치
pip install -r requirements.txt

# 2) 대시보드 실행 (DB 파일이 있는 디렉터리에서)
streamlit run app.py
```

- Python 3.9+ 권장
- `쉬었음 청년 분석.db`가 `app.py`와 같은 디렉터리에 있어야 합니다.

### 의존성
`streamlit`, `pandas`, `plotly`, `numpy`, `scipy`
