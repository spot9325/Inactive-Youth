## 어떤 식으로 분석하는가 (전처리 → 파생변수)

원본 변수만으로는 "격차"가 안 보이므로, `app.py`가 4가지 파생 지표를 직접 만들어냅니다.

**1) 범주형 플래그** — 0/1 숫자를 "부모 동거 / 비동거", "가족 도움 가능 / 없음", "도움망 있음 / 없음" 같은 해석 가능한 라벨로 변환.

**2) 위험점수 (0~5점)** — 취약 요인 5개가 몇 개나 겹치는지 합산:

```179:191:app.py
df_survey["위험점수"] = (
    (df_survey[COL_PARENT] == 0).astype(int) +
    (df_survey[COL_FAMILY] == 0).astype(int) +
    (df_survey[COL_NONE] == 1).astype(int) +
    (df_survey[COL_DEBT] > 0).astype(int) +
    (df_survey[COL_INTEREST] > 0).astype(int)
)
```

점수에 따라 저위험(0~1) / 중위험(2~3) / 고위험(4~5)으로 분류합니다. 즉 "소득이 낮다"가 아니라 **위험요인이 중첩됐는지**를 핵심으로 봅니다.

**3) 생활안전망 유형 (6가지)** — 우선순위 규칙으로 각 청년을 하나의 유형에 배정:

```193:206:app.py
def classify_type(row):
    if row[COL_NONE] == 1:
        return "고립위험형"
    if row[COL_DEBT] > 0 or row[COL_INTEREST] > 0 or row[COL_LIVING_DEBT] > 0:
        return "금융부담형"
    if row[COL_FAMILY] == 1 and row[COL_PARENT] == 1:
        return "가족완충형"
    if row[COL_PUBLIC] == 1:
        return "공공지원형"
    if row[COL_FRIEND] == 1 or row[COL_PRIVATE] == 1:
        return "대체지원형"
    return "취약잠재형"
```

**4) 생활비 구간** — 100만 원 미만 / 100~200 / 200~300 / 300만 원 이상.

## 페이지별 분석 흐름 (스토리텔링 7단계)

사이드바에서 유형·위험수준·부모동거·부채 등 9개 필터를 걸면 모든 페이지가 그 부분집합으로 다시 계산됩니다.

1. **전체 규모와 문제 제기** — 연도별 추이 선그래프 + 대상 수/평균 생활비/평균 부채/평균 위험점수 KPI. "규모만 보지 말고 내부 격차를 보자"는 출발점.
2. **무엇으로 버티는가** — **Sankey 다이어그램**으로 사적·공적 이전소득, 본인 소득, 생활비 부채 → 생활비 지출로 이어지는 자금 흐름을 시각화. 막대그래프로 자원 총량 비교.
3. **가족 안전망** — 부모 동거별 부채 박스플롯, 가족지원별 생활비 박스플롯, 그리고 부모동거→가족지원→부채여부 **Sunburst**.
4. **가족 밖의 안전망** — 가족 도움 없는 집단만 떼어내 지인/공공/민간/도움없음 분포, 공적소득-부채 히트맵, 생활비-소득-부채 산점도로 "가족 없으면 어디로 가는가" 추적.
5. **위험지수 분석** — 평균 위험점수 **게이지 차트**, 위험점수 히스토그램, 위험수준별 부채 박스플롯, 위험수준별 특성 요약표.
6. **생활안전망 유형** — 유형별 규모 **Treemap**, 생활비-소득-부채 산점도(색=유형), 유형별 평균값 프로파일 표.
7. **최종 결론** — 안전망 구성 비율 막대그래프 + **레이더 차트**로 가족/지인/공공/민간/비고립 수준을 종합. 정책 제안으로 마무리.

## 분석 관점의 특징

- 모든 결측값(NaN)을 0으로 처리하고, 필터 결과가 비면 경고문을 띄우는 방어 코드가 있습니다(`safe_mean`, `safe_sum`).
- 각 페이지마다 `insight_box`(해석) + `policy_box`(정책 시사점)가 붙어, 단순 시각화가 아니라 **정책 보고서 형식**으로 결론을 끌어냅니다.
- 핵심 메시지: 쉬었음 청년 문제를 "취업 여부"가 아니라 **"생계 유지 기반(가족·공적지원·부채·고립)의 격차"** 문제로 재정의합니다.

## C. 분석 방법이 잘못된 부분

### 3) 페이지4 "공적 이전소득과 부채의 관계" 히트맵 ― 통계적으로 관계 없음 ❌

실제 상관: **Pearson r = 0.018 (p = 0.56), Spearman = 0.011 (p = 0.72)** → 두 변수는 **사실상 무상관**입니다. 게다가 공적이전소득 85.7%·부채 80.5%가 0이라 히트맵은 원점에 점이 뭉쳐 아무 패턴도 안 보입니다. "관계"라는 제목·해석은 데이터가 뒷받침하지 못합니다.

### 4) 페이지5 위험점수 ― "부모 비동거" 항목이 아무에게도 안 더해지는 버그 ❌ (치명적)
```179:185:app.py

        (df_survey[COL_PARENT] == 0).astype(int) +   # ← 버그
        ...
```
`부모 동거 여부`는 실제로 **1(동거)/2(비동거)** 로 코딩돼 있고 **0은 단 한 명도 없습니다**. 그런데 코드는 `== 0`을 검사하므로:
- 비동거 청년 **291명** 전원이 이 위험요인 점수를 못 받습니다.
- 위험점수 만점은 설계상 5점이지만 **실제 최대는 4점**, 분포는 0:715 / 1:146 / 2:168 / 3:17 / **4:16명(1.5%)**.
- 페이지5 게이지는 `range [0,5]`, 고위험 구간을 [3,5]로 잡지만 5점은 구조적으로 불가능 → 고위험이 실제보다 축소됩니다.

→ `(df_survey[COL_PARENT] == 2)` 또는 `(df_survey[COL_PARENT] != 1)` 로 고쳐야 합니다. (참고로 라벨 생성·`classify_type`은 `==1` 기준이라 정상 작동합니다. 버그는 위험점수에만 있습니다.)

### 5) 페이지3 가족 안전망 ― 해석 방향이 데이터와 반대일 수 있음 ⚠️

## 왜 해야 하나

### 1) 0-팽창(zero-inflation) 변수에 "평균"은 오해를 부른다
제가 검증한 실제 분포입니다.

| 변수 | 0인 비율 | 평균 | 중앙값 | 최대 |
|---|---|---|---|---|
| 청년 부채 총액 | 80.5% | 745만 | **0** | 5억(50000만) |
| 월평균 이자 | 86.3% | 5만 | 0 | 900만 |
| 생활비 부채 | 96.0% | 14만 | 0 | 2000만 |
| 공적 이전소득 | 85.7% | 48만 | 0 | 1600만 |

문제는 "평균 부채 745만 원"이라고 쓰면 독자는 *"쉬었음 청년이 평균적으로 745만 원 빚이 있구나"* 로 읽습니다. 하지만 실제로는 **10명 중 8명은 빚이 0이고**, 빚이 있는 소수(19.5%)가 평균 약 3,800만 원을 지고 있어서 그 소수가 전체 평균을 끌어올린 겁니다. → "평균"이 **누구도 대표하지 못하는 값**이 됩니다.

→ 정직한 표현은 **두 숫자로 분리**하는 것입니다:
- **보유율**: "쉬었음 청년의 19.5%가 부채 보유"

## 어떤 검정을 쓸지 (제안)

데이터 특성이 **0-팽창 + 강한 왜도(부채 최대 5억) + 소규모 이진 그룹**이라, 정규성 가정하는 t-test는 부적절합니다. → **비모수 검정** 위주로 갑니다.

검정은 **순환논리(circularity)** 를 피하는 게 핵심입니다. 위험점수·생활안전망유형은 부채·이자·가족 변수로 **만들어진** 파생변수라, 그걸 다시 부채로 검정하면 "당연히 유의"한 가짜 결과가 나옵니다. 그래서 **독립적인 원변수 관계**만 검정합니다.

| # | 주장(페이지) | 검정 | 이유 |
|---|---|---|---|
| 1 | 부모동거 ↔ 부채 보유 여부 (P3) | **카이제곱 독립성 검정** (2×2) | 보유율(이진) 비교에 적합, 해석 쉬움 |
| 2 | 부모동거 ↔ 부채 총액 (P3) | **Mann–Whitney U** | 부채가 왜도·0팽창 → 중앙값/분포 비교 |
| 3 | 가족지원 ↔ 월 생활비 (P3) | **Mann–Whitney U** | 생활비 분포 비교, 비모수 |
| - | 공적이전 ↔ 부채 (P4) | 이미 Spearman 표기됨 | 유지 |

각 검정 옆에 **p값 + 그룹별 중앙값/보유율 + n**을 같이 표기하고, **사이드바 필터로 한 그룹이 비거나 표본이 너무 작으면(예: n<10) 검정을 자동 생략**하는 가드를 넣겠습니다(필터 때문에 깨지지 않게).

**제외**: 페이지5(위험수준)·6(유형) 검정 — 변수 자체가 부채/가족으로 정의돼 순환논리라 일부러 안 합니다(이 점도 캡션에 명시 가능).

## 근데 당신 말도 맞아요

"공적지원 ↔ 부채 무관"이라는 null 자체는 보여줄 가치가 있어요. 지금 막대그래프(노출 비율)는 그 메시지를 *간접적으로*만 담고 있고, 솔직히 "관계 없음"을 **직접** 보여주진 않죠.

그래서 정말 "관계 없음"을 시각적으로 보여주고 싶다면, 0-팽창에 강한 방식이 따로 있어요:

- **2×2 비교**: 공적지원 보유/미보유 집단의 **부채 보유율**을 나란히 막대로 → 두 막대 높이가 비슷하면 "관계 없음"이 한눈에 보임 (+ 카이제곱 p값). 이게 null을 가장 깔끔하게 보여줍니다.
- **보유자만 산점도**: 둘 다 >0인 사람만 추려 scatter + 추세선 → 0 덩어리에 안 가려짐.
- **히트맵 복원 + 정직한 제목**: "관계 없음 (대부분 0)"으로 라벨만 바로잡아 부활.
